# Portfolio Work Log

다른 채팅에서 포트폴리오 작업을 이어갈 때 참고하기 위한 작업 기록입니다.

## 현재 작업 기준

- 기본 노출 프로젝트는 2개로 유지합니다. `MAX_VISIBLE_PROJECTS = 2` 구조는 변경하지 않습니다.
- 사용자가 작업관리에서 원하는 프로젝트 2개를 선택할 수 있는 구조를 유지합니다.
- FlexiRoute는 포트폴리오 내용에 포함하지 않습니다.
- 웹 본문을 먼저 정리한 뒤, 마지막에 PDF 화면(`src/pages/pdf/PdfPortfolio.tsx`)을 웹 기준에 맞춥니다.
- 없는 역할이나 성과를 새로 만들지 않고, README/Wiki/프로젝트 파일에 근거가 있는 내용만 다듬습니다.
- 이력서에서 정한 기준을 포트폴리오 문장에도 이어갑니다.

## 문장 수정 기준

- `했습니다` 반복처럼 보이는 문장보다, 문제-조치-결과-효과가 드러나는 문장을 우선합니다.
- 성과 수치는 가능한 경우 전후 수치와 개선율을 함께 적습니다.
- 수치 뒤에는 "그래서 어떤 효과가 있었는지"를 한 문장으로 연결합니다.
- `SQL 실행`, `SQL Count`처럼 보는 사람이 무엇을 의미하는지 헷갈릴 수 있는 표현은 피합니다.
- DB 조회 관련 성과는 `이벤트 목록 요청 1회당 DB 조회 쿼리 수`, `반복 DB 조회 쿼리`처럼 맥락을 붙여 씁니다.
- 기준값이 없는 `0건`에는 임의로 퍼센트를 붙이지 않습니다.
- 체크 표시 아이콘은 AI 티가 난다는 인상이 있어 사용하지 않습니다.

## 구조 기준

  - Projects 제목 바로 아래의 프로젝트 간략 소개 카드 기본 정보 담당
  - `badge`, `name`, `description`, `tech`, `role`, `challenge`
- 각 프로젝트 파일의 `overviewSummary`

  - 프로젝트 간략 소개 카드 레이아웃 담당
  - FeedShop/3M 카드의 `소개`, `성과`, `회고`, `담당 역할`, `핵심 과제` 행 위치를 맞춘 상태
- `src/pages/home/components/AboutSection.tsx`
