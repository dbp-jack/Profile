- `했습니다` 반복처럼 보이는 문장보다, 문제-조치-결과-효과가 드러나는 문장을 우선합니다.

- 기준값이 없는 `0건`에는 임의로 퍼센트를 붙이지 않습니다.

## 최근 수정 내용

- FeedShop
  - `6,818ms -> 638ms`에 `약 91% 단축`을 추가했습니다.
  - 0630 이력서 기준으로 조회 쿼리 성과는 `42회 -> 2회 + Redis 캐시 적용`으로 정리합니다.
  - `42회 -> 0회`, `100% 감소`, `SQL Count`는 요약/첫 노출 지표에서 사용하지 않습니다.
  - 성과 효과는 `재방문 진입점의 사용자 이탈 방지`, `동시 3,000명 투표 HTTP 오류·DB 중복 저장 0건`, `투표 수 정합성 확보`로 정리합니다.
  - 캐시 무효화 관련 설명은 사용자가 별도 작업 후 다시 이야기하기로 했으므로, 현재 본문에서는 과하게 확장하지 않습니다.
- 3M
  - 성과는 `UserService→Auth 결합도 0건`, `순환 의존 없음`, `MASTER/HUB_MANAGER/미인증 응답 검증`을 한 문장으로 묶습니다.
  - 성과 효과는 `인증 변경 영향 범위 축소`, `권한 판단 기준 통일`입니다.
  - `일반 권한 판단`은 모든 권한 검사를 뜻하지 않습니다. Gateway가 JWT의 `userId·role`을 검증해 처리할 수 있는 일반 요청의 인증/인가 판단을 뜻합니다.
  - 이 표현의 핵심은 매 요청마다 User 서비스를 재조회하지 않고, JWT payload 기반 사용자 컨텍스트를 `X-User-*` 헤더로 전달해 일반 인증 경로를 처리했다는 점입니다.
  - 사용자 상세 정보가 필요하거나 `@RequiresMasterRole` 같은 명시적 역할 검증이 필요한 경우에는 User 호출 또는 AOP 권한 검증으로 이어지는 구조로 설명합니다.
- FIX
  - 상세 본문 수정은 보류합니다. FeedShop/3M 상세 본문과 PDF 정합성 확인을 먼저 진행한 뒤 별도 작업으로 다시 봅니다.
  - 성과는 `Kafka 이벤트로 주문 생성과 결제 처리 분리`, `결제 실패·취소를 주문 취소 보상 흐름으로 연결`을 한 문장으로 묶습니다.
  - 성과 효과는 `결제 장애 전파와 상태 불일치 위험 완화`입니다.
- About
  - 하이라이트는 0630 이력서 기준으로 `응답시간 91% 단축 · 조회 쿼리 42회 -> 2회` 기준으로 정리했습니다.
