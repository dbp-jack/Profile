## 문장 수정 기준

- `했습니다` 반복처럼 보이는 문장보다, 문제-조치-결과-효과가 드러나는 문장을 우선합니다.
- 성과 수치는 가능한 경우 전후 수치와 개선율을 함께 적습니다.
- 수치 뒤에는 근거로 확인한 기술적 효과를 연결합니다. 운영 데이터가 없으면 이탈·재방문·매출 개선을 성과로 단정하지 않습니다.
- `SQL 실행`, `SQL Count`처럼 보는 사람이 무엇을 의미하는지 헷갈릴 수 있는 표현은 피합니다.
- DB 조회 관련 성과는 `이벤트 목록 요청 1회당 DB 조회 쿼리 수`, `반복 DB 조회 쿼리`처럼 맥락을 붙여 씁니다.
- 기준값이 없는 `0건`에는 임의로 퍼센트를 붙이지 않습니다.
- 체크 표시 아이콘은 AI 티가 난다는 인상이 있어 사용하지 않습니다.
- PDF에서 기존 `13.8px` 설명 본문은 `14.6px`로 한 단계 확대하고, 같은 크기를 쓰던 제목·링크 라벨은 기존 크기를 유지합니다.
